# 📊 User Progress Monitoring Implementation Plan

ইউজারদের পড়ার অগ্রগতি (Reading Progress) এবং কুইজ পারফরম্যান্স ট্র্যাক করার জন্য একটি কার্যকর মনিটরিং সিস্টেম তৈরির পরিকল্পনা নিচে দেওয়া হলো।

## ১. পড়ার অগ্রগতি (Reading Progress) সিংক্রোনাইজেশন
বর্তমানে "পড়েছি" (Mark as Read) ডাটা শুধুমাত্র ব্রাউজারের `LocalStorage`-এ থাকে। এটি ডাটাবেসে নিয়ে আসতে হবে।

- [ ] **Database Change:** `user_progress` টেবিলে `read_questions_count` বা `read_questions_ids` (JSON) কলাম যোগ করা। অথবা আলাদা একটি `question_user_reads` পিভট টেবিল তৈরি করা।
- [ ] **API Endpoint:** একটি নতুন রাউট তৈরি করা যা ফ্রন্টএন্ড থেকে পড়ার প্রগ্রেস সেভ করবে।
- [ ] **Frontend Update:** `questions.blade.php` এর JS ফাংশন আপডেট করা যাতে LocalStorage এর পাশাপাশি ডাটাবেসেও আপডেট সেন্ড হয়।

## ২. অ্যাডমিন মনিটরিং ইন্টারফেস (Admin Dashboard)
অ্যাডমিন যাতে প্রতিটি ইউজারের প্রগ্রেস দেখতে পারে তার জন্য নতুন ফিচার যোগ করা।

- [ ] **User Details Page:** এডমিন প্যানেলে ইউজার লিস্ট থেকে কোনো ইউজারে ক্লিক করলে তার বিস্তারিত প্রোফাইল দেখা যাবে।
- [ ] **Stats to Show:**
    - কোন ক্যাটাগরিতে কতটি প্রশ্ন পড়েছে।
    - কোন লেভেলের কুইজে কত স্কোর পেয়েছে এবং কততম চেষ্টায় পাস করেছে।
    - সবশেষ কখন লগিন করেছে।

## ৩. ইউজার প্রোফাইল (Student Dashboard)
শিক্ষার্থীরা যেন নিজের প্রগ্রেস নিজেই দেখতে পারে।

- [ ] **Student Profile View:** ফ্রন্টএন্ডে একটি প্রোফাইল পেজ যেখানে ইউজার তার অর্জিত মেডেল বা কমপ্লিটেড লেভেল দেখতে পাবে।

- [ ] ডাটাবেসে রিড প্রগ্রেস সেভ হচ্ছে কি না তা চেক করা।
- [ ] এডমিন প্যানেল থেকে ইউজারের প্রোফাইল এবং কুইজ হিস্ট্রি দেখা যাচ্ছে কি না তা যাচাই করা।
