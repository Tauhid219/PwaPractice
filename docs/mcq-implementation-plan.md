# 📝 MCQ Implementation Plan (3 Options)

 কুইজ সিস্টেমকে শর্ট-অ্যানসার থেকে ৩-অপশন বিশিষ্ট MCQ ফরম্যাটে রূপান্তরের পরিকল্পনা।

## ১. ডাটাবেস পরিবর্তন (Database Schema)
প্রশ্নের অপশনগুলো সংরক্ষণের জন্য `questions` টেবিল আপডেট করা।

- [ ] **Migration:** `add_options_to_questions_table` মাইগ্রেশন তৈরি করা।
    - `option_1`, `option_2`, `option_3` (String) কলাম যোগ করা।
    - `answer_text` কলামটিকে সঠিক অপশন (e.g., ওয়ান, টু, বা থ্রি) বুঝানোর জন্য ব্যবহার করা অথবা এটি আগের মতোই রাখা। 
    - *পরামর্শ:* `answer_text` এ সরাসরি সঠিক উত্তরটিই থাকবে এবং এটি `options` এর একটির সাথে মিলতে হবে।

## ২. অ্যাডমিন প্যানেল আপডেট (Admin CRUD)
- [ ] **Question Controller & Request:** ৩টি অপশন ইনপুট নেয়ার ব্যবস্থা করা এবং ভ্যালিডেশন যোগ করা।
- [ ] **Views:** ক্রিয়েট এবং এডিট ফর্মে ৩টি নতুন টেক্সট বক্স যোগ করা।

## ৩. এক্সেল ইম্পোর্ট আপডেট (Excel/CSV Import)
- [ ] **QuestionImport.php:** ৩টি কলাম (Option 1, Option 2, Option 3) থেকে ডাটা রিড করে ডাটাবেসে সেভ করার লজিক আপডেট করা।

## ৪. ফ্রন্টএন্ড কুইজ ইন্টারফেস (Frontend UI)
- [ ] **taking.blade.php:** 
    - টেক্সট ইনপুটের বদলে ৩টি রেডিও বাটন (Radio Buttons) বা কার্ড-স্টাইল বাটন যোগ করা।
    - সিলেক্ট করার পর অটোমেটিক নেক্সট বাটনে যাওয়ার সুবিধা রাখা যেতে পারে।

## ৫. কুইজ সাবমিশন লজিক (Backend Validation)
- [ ] **QuizController.php:** সাবমিট করা অপশনটির সাথে ডাটাবেসের `answer_text` এর তুলনা করা (Trim and Lowercase insensitive)।
